# NOLYK SHOP Telegram Bot

Многофункциональный Telegram-магазин на `aiogram 3.28` и SQLite.

## Что уже разложено по структуре

- Главное меню с единым баннером и редактируемыми текстами.
- Профиль, баланс, пополнение, промокоды, история покупок.
- Каталог с категориями и товарами из админ-панели.
- Покупка товара с автосписанием баланса и логом администратору.
- Пополнение через карту с заявкой, скрином и кнопками принять/отклонить.
- Опциональная интеграция CryptoBot/CryptoPay через API токен.
- Админ-панель: настройки, каталог, пользователи, статистика, рассылка, обязательные подписки.
- Лог-чат для покупок, пополнений, новых пользователей и действий.
- Inline-навигация редактирует текущее сообщение через `edit`, а не кидает новую панель на каждый клик.

## Где указать баннер

Откройте `/admin` → `Оформление` → `Баннер` и вставьте прямую ссылку на картинку:

```text
https://example.com/banner.jpg
```

Этот баннер используется как общий баннер меню, профиля, каталога и других экранов.

## Рассылка

Откройте `/admin` → `Рассылка`.

В конструкторе есть 3 основные кнопки:

- `Фото` — загрузить или очистить фото рассылки.
- `Текст` — задать текст рассылки с Telegram-форматированием.
- `Inline кнопки` — добавить кнопки по одной: текст, ссылка, стиль.

Стили `default`, `primary`, `success`, `danger` сохраняются в структуре кнопки. Обычные Telegram inline-кнопки через Bot API не имеют поля цвета/стиля, поэтому визуально Telegram покажет стандартные inline-кнопки.

## Админ-разделы

- `Оформление` — общий баннер и тексты главного меню, профиля, FAQ, инфо и платежных сценариев.
- `Платежи` — карта, ФИО, банк и курс RUB.
- `Настройки` — username владельца и чат логов.

## Запуск

1. Создайте виртуальное окружение и установите зависимости:

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

2. Скопируйте `.env.example` в `.env` и заполните:

```bash
copy .env.example .env
```

3. Запустите бота:

```bash
python -m app.main
```

## Важно

Telegram Bot API не поддерживает цветные inline-кнопки (`danger`, `primary`, `success`) и custom emoji внутри текста inline-кнопок как entity. В сообщениях custom emoji и жирный текст сохраняются через Telegram HTML/entities, а в кнопках остается обычный видимый текст.
