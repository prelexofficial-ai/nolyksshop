# Структура проекта

```text
app/
  main.py                 точка запуска бота
  config.py               загрузка .env и владельцев
  database.py             SQLite-схема и методы работы с данными
  filters.py              проверка админ-доступа
  states.py               FSM-состояния сценариев

  handlers/
    user/main.py          пользовательское меню, каталог, покупки, пополнения
    admin/main.py         админ-панель, настройки, каталог, пользователи, рассылка

  keyboards/
    user.py               inline-кнопки пользователя
    admin.py              inline-кнопки админки

  services/
    crypto_pay.py         CryptoBot/CryptoPay API
    logs.py               отправка событий в лог-чат
    screens.py            единый баннер на экранах
    subscriptions.py      проверка обязательных подписок

  utils/
    formatting.py         HTML, суммы, коды покупок, карточки товаров
```

## Основной поток

1. Пользователь пишет `/start`.
2. Бот создает пользователя в базе и проверяет блокировку.
3. Если включены обязательные подписки, бот проверяет каналы.
4. После доступа открывается главное меню с единым баннером.
5. Пользователь открывает профиль, каталог, инфо или FAQ.
6. Покупка списывает баланс, генерирует код и отправляет лог.
7. Пополнения через карту уходят админу на подтверждение.

## Админ-панель

Открывается командой `/admin` или кнопкой в главном меню владельца. Доступ берется из `OWNER_IDS` в `.env`.
